# Blocky13 – Project Summary

## Overview
**Blocky13** is a Fabric mod for Minecraft 1.21.11 that adds 150 derivative block types across various material bases (wood, stone, metal, etc.), including slabs, stairs, fences, gates, doors, trapdoors, pressure plates, buttons, chains, and iron bars. The mod is data-driven and heavily templated, using Python scripts to generate resource files (models, textures, blockstates, recipes, loot tables, and advancements).

**Mod ID:** `blocky13` | **Version:** 1.0.0 | **Java:** 21 | **Minecraft:** 1.21.11

---

## Architecture

### Code Structure
- **ModBlocks.java** – Central registry: maintains a `BASES` table (base name → vanilla Block) that drives procedural generation of all 150 block variants (10 types × 15 bases).
- **Blocky13.java** – Mod entry point; initializes ModBlocks and populates CUTOUT_BLOCKS for client rendering.
- **Blocky13Client.java** – Client-side setup; applies cutout render layer to transparent blocks (bars, chains).
- **ExampleMixin.java** – Placeholder; can be extended for custom block behavior patches.

### Data-Driven Generation
Resource files (blockstates, models, textures, recipes, loot tables, advancements) are **generated by Python scripts**:
- Vanilla block definitions are templated via prefix-replace (e.g., `minecraft:block/oak_stairs` → `blocky13:block/<name>_stairs`).
- Chain and bars textures are programmatically recolored from vanilla `iron_chain`/`iron_bars` PNGs to match each material's base color (using Pillow; luminance-ratio tinting preserves shape).
- Recipes, loot tables, and advancement unlock data are generated with correct singularity (e.g., `data/<ns>/recipe/`, not `recipes/`).

```

├── src/main/java/com/blocky13/
│   ├── Blocky13.java              (mod entry + main init)
│   ├── Blocky13Client.java        (client rendering setup)

```

## Key Features & Implementation Notes

### 1. Redstone Power (Powered Variants)
Five block types can power redstone circuits when crafted from `redstone_block` material:
- **Pressure Plates / Buttons:** Vanilla components; already emit power by default.
- **Slabs / Stairs / Fence / Gate / Door / Trapdoor / Chain / Bars:** Custom `Powered*Block` subclasses override:
  - `isSignalSource()` → `true`
  - `getSignal()` → `15`

### 2. Transparency & Rendering
Blocks with cutout textures (bars, chains) are registered client-side to the `CUTOUT` render layer:
```java
BlockRenderLayerMap.putBlock(block, ChunkSectionLayer.CUTOUT);
```
Without this, the blocks render as opaque and are visually incorrect.

### 3. Fence Connectivity
Custom fences connect via block tag `minecraft:fences` (not instanceof):
- Tag file: `data/minecraft/tags/block/fences.json`
- Custom fences **excluded** from `wooden_fences` so they prefer connecting to each other.
- Iron bars connect via vanilla instanceof (no tag needed).

### 4. Recipe Unlocks
**Critical:** A recipe without an unlock advancement **never appears in the recipe book**, even though crafting works manually. Solution:
- Per-recipe advancement at `data/<ns>/advancement/recipes/<group>/<name>.json`
- Criteria: `inventory_changed` (on ingredient) + `recipe_unlocked` (on recipe)
- Parent: `minecraft:recipes/root`
- Rewards: unlock the recipe via `rewards.recipes`

All 150 recipes have auto-generated advancements.

### 5. Minecraft 1.21.11 Gotchas (Fabric Mojang Mappings)
- **Block/Item ID requirement:** Must call `.setId(ResourceKey.create(...))` before construction; crashes with `NullPointerException: Block id not set` otherwise.
- **Identifier, not ResourceLocation:** Use `net.minecraft.resources.Identifier.fromNamespaceAndPath(...)`.
- **Data directory singularity:** Use `recipe/` and `loot_table/` (not plural); `advancement/` is singular; the singular forms are loaded, plurals silently ignored.
- **Item texture model:** Items need `assets/<ns>/items/<name>.json` with model reference; missing files render as missing-texture with no error log.
- **Recipe ingredient format:** Use plain strings `"#": "minecraft:dirt"`, NOT object form `{"item": "minecraft:dirt"}`.

## Known Issues / Context

- **Sibling mods:** User also maintains `minecraft-food-mod` (id `food-by-giamat`) and `more-totems`. The food mod's popsicle/tray recipes still use the old `{"item": "..."}` ingredient format (pre-1.21.11) and fail to parse; blocky13 was corrected in a recent commit.
- **Recipe collision caveat:** Shaped recipes with identical patterns and ingredients will conflict if they have different outputs. The 15 bases use distinct grid patterns to avoid collisions.
